# BTC 事件驱动短线交易系统

创建时间：2026-06-03

> 目的：把盘中观察到的「止损猎杀、空头爆仓、盘口买卖压、主动成交、OI 变化」沉淀成一套可执行的 BTC 短线交易框架。

---

## 1. 核心思想

这套系统不预测涨跌，而是等待市场出现高确定性的事件：

1. **主动成交翻转**：taker buy / taker sell 明显变化。
2. **盘口失衡**：±0.1%、±0.25%、±0.5% 买卖盘比例变化。
3. **OI 持仓变化**：判断是新仓进场、止损、爆仓还是去杠杆。
4. **价格结构确认**：假跌破、假突破、回踩是否守住。

核心原则：

> 只做「扫损、爆量、买卖盘翻转、OI 变化」后的确认交易，不做中间震荡。

---

## 2. 关键指标

### 2.1 主动买盘强度

```text
buy_ratio = taker_buy_quote / total_quote_volume
delta = taker_buy_quote - taker_sell_quote
```

判断：

```text
buy_ratio > 0.60  = 强买盘
buy_ratio < 0.40  = 强卖盘
0.45–0.55         = 噪音区
```

---

### 2.2 盘口强度

```text
orderbook_imbalance = bid_depth / (bid_depth + ask_depth)
```

重点观察：

- ±0.10%
- ±0.25%
- ±0.50%

判断：

```text
> 0.58 = 买盘明显占优
< 0.42 = 卖盘明显占优
0.45–0.55 = 平衡区
```

---

### 2.3 爆量判断

```text
volume_ratio = 当前1分钟成交量 / 最近20分钟平均成交量
```

判断：

```text
> 2 = 爆量
> 3 = 异常爆量
```

---

### 2.4 OI 判断

```text
oi_change = 当前OI / N分钟前OI - 1
```

解释：

```text
价格上涨 + OI下降 = 空头平仓 / short squeeze
价格上涨 + OI上升 = 新多进场
价格下跌 + OI下降 = 多头止损 / 去杠杆
价格下跌 + OI上升 = 新空进场
```

---

## 3. 交易模式

## 模式 A：扫损反转多

适合场景：价格先快速跌破短线低点，触发止损后又快速收回。

### 触发条件

```text
1. 价格跌破最近20–30分钟低点
2. 跌破时 1m buy_ratio < 0.30
3. 1m成交量 > 最近20分钟均量 * 2
4. 之后3分钟内价格重新站回跌破位
5. 收回时 buy_ratio > 0.60
6. OI下降或持平
```

### 入场

价格重新站回被跌破位置时做多。

### 止损

```text
止损 = 扫损低点下方 0.10%–0.20%
```

### 止盈

```text
第一目标：前震荡区高点
第二目标：上方大卖墙
第三目标：1–1.5R 或短线 ATR 目标
```

---

## 模式 B：空头爆仓后的回踩多

适合场景：短时间暴涨后，不追第一根大阳线，等待回踩确认。

### 触发条件

```text
1. 10分钟涨幅 > 0.50%
2. 10分钟合约净主动买入 > 0
3. 合约 buy_ratio > 0.55
4. OI下降，确认空头回补
5. 回踩不破突破位 / 拉升起点 / 关键支撑
6. 回踩时盘口买盘占比 > 0.55
```

### 入场

回踩突破位附近，且买盘仍然强时做多。

### 止损

```text
止损 = 回踩低点下方
```

### 止盈

```text
第一目标：前高
第二目标：上方卖墙
第三目标：整数关口 / 高级别压力
```

---

## 模式 C：大卖墙失败短空 / 多单止盈

适合场景：价格冲到上方大卖墙，主动买盘衰减，价格多次冲不上去。

### 触发条件

```text
1. 价格接近上方大卖墙
2. 连续2–3次冲不上去
3. buy_ratio 从 >0.60 降到 <0.50
4. 合约盘口 ask_depth 明显大于 bid_depth
5. OI没有继续增加
```

### 操作

```text
已有多单：减仓 / 止盈
激进短线：小仓短空
```

### 止损

```text
止损 = 卖墙上方
```

### 止盈

```text
第一目标：震荡中轴
第二目标：下方买墙
第三目标：前低
```

---

## 4. 不交易条件

以下情况不做：

```text
1. 现货买盘强，但合约盘口卖压极重
2. OI快速下降，价格却横盘
3. 主动买卖频繁反转
4. 价格卡在大买墙和大卖墙中间
5. 第一根暴涨K线刚出现，不追
6. buy_ratio 位于 0.45–0.55 噪音区，且盘口无明显优势
```

---

## 5. 风控规则

### 5.1 单笔风险

```text
每笔亏损不超过本金 0.3%–1.0%
```

### 5.2 必须提前设止损

不允许临时主观移动止损。

### 5.3 不追第一根大阳线

暴拉之后只做两种：

```text
1. 回踩做多
2. 冲不上去做空 / 止盈
```

### 5.4 连续亏损暂停

```text
连续亏2笔：暂停30分钟
当天亏损超过2%：停止交易
```

### 5.5 不在盘口夹层交易

例子：

```text
下方买墙：67,000
上方卖墙：67,200
价格在 67,100 中间时，不做。
```

---

## 6. 实盘执行流程

每分钟执行一次：

```text
1. 获取现货 / 合约 1m K线
2. 获取现货 / 合约盘口深度
3. 计算主动买卖量、buy_ratio、delta
4. 获取 OI 变化
5. 计算盘口 imbalance
6. 判断是否触发模式 A / B / C
7. 满足条件则下单，否则观望
8. 下单后立即设置止损和分批止盈
9. 记录每笔交易和触发理由
```

## 7. 盘中案例：2026-06-03 BTC 暴涨

观察到的结构：

```text
17:48  价格被砸到 66,656，buy_ratio 只有 14%，疑似止损猎杀
17:49  爆量 162.9 BTC，价格拉到 67,037，buy_ratio 78%
17:50  继续拉到 67,210，buy_ratio 70%
17:51–17:52  最高约 67,322，随后出现获利了结
```

解释：

```text
先砸穿止损密集区 → 触发多头止损 / 空头追空 → 随后主动买盘爆发 → 空头被迫回补 → short squeeze
```

对应模式：

```text
模式 A：扫损反转多
模式 B：空头爆仓后的回踩多
```

关键结论：

> 这不是普通上涨，而是「止损猎杀 + 空头爆仓 + 主动买盘翻转」。

## 8. 当前使用原则

```text
1. 只在关键位附近交易
2. 只在信号共振时交易
3. 盘口、主动成交、OI 至少两项同向才交易
4. 有大墙就先观察是否被吃掉，不提前猜
5. 若大墙被吃掉且 buy_ratio > 0.60，可顺势
6. 若大墙前 buy_ratio 跌破 0.50，可止盈或反手小仓
```

---

## 9. 一句话总结

> 这套 BTC 短线系统的核心是：等待扫损或爆量事件出现，再用主动成交、盘口深度和 OI 变化确认方向，做确认后的跟随，而不是主观猜方向。
